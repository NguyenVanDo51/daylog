# E2E Testing with Maestro — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Maestro E2E tests covering three core user journeys (smoke, day grid, story viewer) runnable locally against the iOS simulator.

**Architecture:** Maestro flows live in `mobile/e2e/flows/`. Auth is bypassed by baking a test JWT into the app at Metro bundle time via `EXPO_PUBLIC_E2E_TEST_TOKEN`. A 3-line guard in `_layout.tsx` short-circuits SecureStore/OAuth when this var is present.

**Tech Stack:** Maestro CLI (global, not a project dependency), Expo 56, React Native 0.85, iOS Simulator

## Task 2: Add E2E auth bypass in `_layout.tsx`

**Files:**
- Modify: `mobile/app/_layout.tsx:43-66`

The existing `useEffect` reads from `SecureStore` to hydrate auth. Add a guard at the top of the async IIFE that short-circuits when `EXPO_PUBLIC_E2E_TEST_TOKEN` is set.

- [ ] **Step 1: Apply the change**

Open `mobile/app/_layout.tsx`. Replace the `useEffect` block (lines 43–66):

```ts
  useEffect(() => {
    (async () => {
      const e2eToken = process.env.EXPO_PUBLIC_E2E_TEST_TOKEN;
      if (e2eToken) {
        setAuth(e2eToken, { id: 'e2e-user', display_name: 'E2E Test', email: 'e2e@test.local', avatar_url: null });
        setReady(true);
        return;
      }
      const stored = await SecureStore.getItemAsync(TOKEN_KEY);
      if (stored) {
        const cachedUser = await SecureStore.getItemAsync(USER_KEY);
        try {
          if (cachedUser) setAuth(stored, JSON.parse(cachedUser));
          const { data } = await api.get('/users/me', { headers: { Authorization: `Bearer ${stored}` } });
          setAuth(stored, data);
          Sentry.setUser({ id: data.id });
          await SecureStore.setItemAsync(USER_KEY, JSON.stringify(data));
          registerPushToken().catch(() => { });
        } catch (err) {
          if ((err as { response?: { status?: number } }).response?.status === 401) {
            await SecureStore.deleteItemAsync(TOKEN_KEY);
            await SecureStore.deleteItemAsync(USER_KEY);
            clearAuth();
            Sentry.setUser(null);
          }
        }
      }
      setReady(true);
    })();
  }, []);
```

- [ ] **Step 2: Type-check**

```bash
cd mobile && npx tsc --noEmit
```

Expected: no errors.

- [ ] **Step 3: Commit**


## Task 4: Create Maestro flow directory and smoke test

**Files:**
- Create: `mobile/e2e/flows/00-smoke.yaml`

The smoke test verifies the app launches without crashing and lands on the Albums screen (authenticated). It asserts that the `menu-btn` testID is visible — this element only renders when `AlbumsPage` is mounted and the app is logged in.

- [ ] **Step 1: Create the directory**

```bash
mkdir -p mobile/e2e/flows
```

- [ ] **Step 2: Create `00-smoke.yaml`**

Create `mobile/e2e/flows/00-smoke.yaml`:

```yaml
appId: com.andynguyen.daylog
---
- launchApp:
    clearState: false
- assertVisible:
    id: "menu-btn"
    timeout: 8000
```

`clearState: false` keeps the Metro bundle cache. `timeout: 8000` gives the app 8 seconds to render — React Query needs a moment to fetch albums.

- [ ] **Step 3: Build the app with the E2E token**

```bash
cd mobile
source .env.e2e
npx expo run:ios
```

This builds the native shell and starts Metro with `EXPO_PUBLIC_E2E_TEST_TOKEN` in the environment. Metro bakes it into the JS bundle. Wait for the simulator to launch and the app to appear before proceeding.

- [ ] **Step 4: Run the smoke test**

In a separate terminal (leave the Metro bundler running):

```bash
cd mobile
maestro test e2e/flows/00-smoke.yaml
```

Expected output:
```
✅  Flow 00-smoke
  ✅  Launch app
  ✅  Assert visible "menu-btn"
```

If the test fails with "Element not found", check that the simulator is running and the app is on screen. Run `maestro hierarchy` to see what's currently visible.

- [ ] **Step 5: Commit**

```bash
git add mobile/e2e/flows/00-smoke.yaml
git commit -m "test(e2e): add smoke flow — app launches authenticated"
```

## Task 5: Day grid flow

**Files:**
- Create: `mobile/e2e/flows/01-day-grid.yaml`

This flow: launches → taps the first album row → asserts the day grid renders (a `day-cell-*` element is visible).

**Pre-condition:** The test account must have at least one album with at least one day of photos. If the account is fresh, add photos manually via the app first.

- [ ] **Step 1: Create `01-day-grid.yaml`**

Create `mobile/e2e/flows/01-day-grid.yaml`:

```yaml
appId: com.andynguyen.daylog
---
- launchApp:
    clearState: false
- assertVisible:
    id: "menu-btn"
    timeout: 8000
- tapOn:
    id: "album-row-.*"
    index: 0
- assertVisible:
    id: "day-cell-.*"
    timeout: 6000
```

`index: 0` taps the first matching element. The `day-cell-.*` regex matches any `day-cell-<date>` testID.

- [ ] **Step 2: Run the flow**

```bash
maestro test mobile/e2e/flows/01-day-grid.yaml
```

Expected output:
```
✅  Flow 01-day-grid
  ✅  Launch app
  ✅  Assert visible "menu-btn"
  ✅  Tap on "album-row-.*" [0]
  ✅  Assert visible "day-cell-.*"
```

If `day-cell-.*` is not found, run `maestro hierarchy` after manually tapping an album to see the testID values rendered on screen.

- [ ] **Step 3: Commit**

```bash
git add mobile/e2e/flows/01-day-grid.yaml
```
